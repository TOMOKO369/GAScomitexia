# 在庫管理システム (Stock Sync) Webアプリ版

Google Apps Script (GAS) と連携して、独自のWebサイト（GitHub Pages等）から在庫数を管理するシステムのコード一式です。

## ファイル構成
- `index.html` : Webサイトの画面（フロントエンド）です。デザインはPremiumなGlassmorphismを採用しています。
- `Code.gs` : Google Apps Scriptに貼り付けるためのバックエンドコードです。

---

## 🚀 設定手順

### ステップ1：GAS（Google Apps Script）の準備
1. Googleドライブを開き、「新規」＞「Google スプレッドシート」を作成します。
2. 上部メニューの「拡張機能」＞「Apps Script」をクリックしてエディタを開きます。
3. エディタに書かれているコードをすべて消し、このフォルダにある `Code.gs` の内容をコピーして貼り付けます。
4. ファイルを保存（Ctrl+S または Cmd+S）します。

### ステップ2：GASを「Webアプリ」として公開（デプロイ）
1. GASエディタの右上にある青い「デプロイ」ボタンを押し、「新しいデプロイ」を選択します。
2. 「種類の選択」の歯車アイコンをクリックし、「ウェブアプリ」を選びます。
3. 以下のように設定します：
   - **説明**: `Stock Sync API` などお好きな名前
   - **実行するユーザー**: `自分`
   - **アクセスできるユーザー**: `全員`
4. 「デプロイ」をクリックします（※初回のみGoogleのアクセス承認画面が出ますので、ご自身のアカウントを選択し、「詳細」＞「～へ移動（安全ではないページ）」＞「許可」と進めてください）。
5. 完了画面で表示される **「ウェブアプリのURL」をコピー** します。

### ステップ3：HTMLファイルの連携とGitHubへの公開
1. このフォルダにある `index.html` をエディタ（VSCode等）やメモ帳で開きます。
2. 下の方にある `<script>` タグ内の以下の行を探します：
   ```javascript
   const GAS_WEB_APP_URL = 'YOUR_GAS_WEB_APP_URL_HERE';
   ```
3. `'YOUR_GAS_WEB_APP_URL_HERE'` の部分を、先ほどコピーした **GASのウェブアプリURL** に置き換えて保存します。

### ステップ4：GitHubでの表示（GitHub Pagesでの公開）
ご自身のGitHubを使ってWebサイトとして表示・公開する手順です。

1. GitHubにログインし、「New repository」から新しいリポジトリを作成します（例: `inventory-app`）。
2. 作成したリポジトリに、URLを書き換えた `index.html` をアップロード（Add file > Upload files）し、Commit（保存）します。
3. リポジトリのメニューバーから「**Settings**」を開きます。
4. 左サイドバーの「**Pages**」をクリックします。
5. 「Build and deployment」セクションの「Branch」を `None` から `main`（または `master`）に変更し、「Save」を押します。
6. 数分待ってからページをリロードすると、「Your site is live at https://あなたのユーザー名.github.io/リポジトリ名/」というURLが表示されます。
7. そのURLにアクセスすると、在庫管理システムのWebサイトが表示されます！スマートフォンからも同じURLでアクセスして利用可能です。

## 🔔 自動メール通知（毎朝8時の在庫チェック）の設定
在庫が「2個以下」になった時に毎朝自動でメール通知を受け取るには、以下の設定を追加で行ってください。

1. GASのエディタ画面に戻ります。
2. 左側のメニューにある時計アイコン（「トリガー」）をクリックします。
3. 右下の「トリガーを追加」ボタンを押します。
4. 以下の通りに設定して「保存」します：
   - **実行する関数**: `checkStockDaily`
   - **イベントのソース**: `時間主導型`
   - **時間ベースのトリガーのタイプ**: `日付ベースのタイマー`
   - **時刻を選択**: `午前 8時～9時`（またはお好きな時間）
